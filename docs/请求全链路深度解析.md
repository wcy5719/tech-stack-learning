### 第4层：业务层（核心处理）

**4.1 Controller层 - 异步处理启动**

```java
@PostMapping("/chat")
public DeferredResult<ResponseEntity<ChatResponse>> chat(ChatRequest request) {
    // 步骤1：创建DeferredResult（占位符）
    DeferredResult<ResponseEntity<ChatResponse>> deferredResult = 
        new DeferredResult<>(600_000L);  // 10分钟超时
    
    // 步骤2：提交异步任务
    CompletableFuture<ChatResponse> future = 
        asyncChatService.processChatAsync(request);
    
    // 步骤3：设置完成回调
    future.whenComplete((response, error) -> {
        if (error != null) {
            deferredResult.setErrorResult(errorResponse);
        } else {
            deferredResult.setResult(ResponseEntity.ok(response));
        }
    });
    
    // 步骤4：立即返回（Tomcat线程释放！）
    return deferredResult;
}
```

## 第四章：关键决策点分析

### 4.1 缓存命中 vs 未命中

```
请求到达
    ↓
布隆过滤器
    ├─ "肯定没有" → 跳过缓存，直接AI推理（节省10ms）
    └─ "可能有"   → 查询缓存
                     ├─ 命中 → <5ms返回 ✨
                     ─ 未命中 → AI推理 ~20s
```

### 4.2 熔断器触发时机

```
vLLM调用失败率统计（最近20次）：
├─ 失败率 < 60% → 继续调用
└─ 失败率 ≥ 60% → 熔断器打开
                   ├─ 直接返回降级回复
                   └─ 30秒后尝试半开状态
```

### 4.3 限流触发

```
IP访问计数（1分钟窗口）：
├─ 计数 ≤ 100 → 放行
└─ 计数 > 100 → 返回429
                 └─ 客户端需等待窗口重置
```
